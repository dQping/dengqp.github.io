本篇主要简单总结一下常用的几种前后端通信方式，包括同源策略的概念和几种常见的跨域通信及实现，所总结的内容不做详细展开。如果需要还需再详细查阅其他资料。

什么是同源策略及限制

同源策略是一个用于隔离潜在恶意文件的关键的安全机制，不同源之间无法做数据交互。

协议、域名和端口相同即为同源。

限制的 3 个方面：

  • Cookie、LocalStorage 和 IndexDB 无法读取
  • DOM 无法获得
  • AJAX 请求不能发送

前后端如何通信

  • AJAX：同源下的通信方式
  • WebSocket：不受同源策略的限制
  • CORS：支持跨域也支持同源

跨域通信常用的几种方式

  • JSONP
  • Hash
  • postMessage
  • webSocket
  • CORS(Cross-origin resource sharing 跨域资源共享)：它允许浏览器向跨源服务器，发出 XMLHttpRequest 请求，从而克服了 AJAX 只能同源使用的限制,可以理解为支持跨域通信的 ajax

ps：(hash 和 search 的区别): hash 就是 url 地址中 # 后面的信息，hash 的改变，页面不会刷；search 是 url 地址中 ？后面的信息，search 的改变会刷新页面。

jsonp 跨域的实现原理解析

实现原理是动态创建 script 标签，请求回来再删除

hash 跨域的实现原理解析

hash 就是 url 地址中 # 后面的信息，hash 的改变，页面不会刷，利用这一点实现跨域通信。

postMessage 跨域的实现原理解析

html5 新增标准，支持跨域。postMessage()方法允许来自不同源的脚本采用异步方式进行有限的通信，可以实现跨文本档、多窗口、跨域消息传递.

更多详细信息，请参考：参考资料

webSocket 跨域的实现原理解析

不受同源策略限制,它的最大特点就是，服务器可以主动向客户端推送信息，客户端也可以主动向服务器发送信息，是真正的双向平等对话，属于服务器推送技术的一种。
更多详细信息，请参考：参考资料

CORS 跨域的实现原理解析:

CORS 需要浏览器和服务器同时支持。目前，所有浏览器都支持该功能，IE 浏览器不能低于 IE10。

整个 CORS 通信过程，都是浏览器自动完成，不需要用户参与。对于开发者来说，CORS 通信与同源的 AJAX 通信没有差别，代码完全一样。浏览器一旦发现 AJAX 请求跨源，就会自动添加一些附加的头信息，有时还会多出一次附加的请求，但用户不会有感觉。因此，实现 CORS 通信的关键是服务器。只要服务器实现了 CORS 接口，就可以跨源通信。
