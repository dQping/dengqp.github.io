react学习-4：使用React生命周期方法

  4. 生命周期方法

React 生命周期主要包括三个阶段：初始化阶段、运行中阶段和销毁阶段，在 React 不同的生命周期里，会依次触发不同的钩子函数。

初始化阶段 Mount

以下说明中带删除线的生命周期方法即将被移除，实际写代码过程中最好不要使用。使用其替代方法

  • constructor()
  • componentWillMount()
  • render()
  • componentDidMount()

运行中阶段 Update

  • componentWillReceiveProps() / static getDerivedStateFromProps()
  • shouldComponentUpdate()
  • componentWillUpdate() / getSnapshotBeforeUpdate()
  • render()
  • componentDidUpdate()

销毁阶段 Unmount

  • componentWillUnmount()

生命周期方法

componentWillMount（即将移除）

componentWillMount()是唯一一个在 render()之前调用的生命周期方法。因此是在服务端渲染中唯一被调用的方法。

注意：因为 componentWillMount()将被删除，所以官方推荐使用 constructor() 替代该方法

componentDidMount

当该方法被调用时候，React 已经渲染了组件并且将组件插入 DOM。因此如有有任何依赖于 DOM 的初始化，应该放在这里。该方法中可以使用 this.setState()方法，它将触发重新渲染。

使用场景：

  • fetch data
  • 依赖 DOM 初始化
  • 添加事件监听

componentWillReceiveProps（即将移除）

当组件接收到新的 props，该方法会首先被调用，但是需要注意一点，即使 props 并没有发生改变，该方法也会被调用，所以使用该方法的时候要确保比较 this.props 和 nextProps，避免不必要的渲染。

注意：componentWillReceiveProps 将被移除，使用新的 static getDerivedStateFromProps 代替。

static getDerivedStateFromProps（新增）

每次 render 都会调用该方法——即使 props 没有发生变化。所以要谨慎使用。

shouldComponentUpdate

该方法在初始化时候不执行。在有些时候需要避免不必要的渲染，可以使用该方法。返回 false 意味着 React 不执行 componentWillUpdate()，render()，componentDidUpdate()。在该方法中不可以设置 setState。

注意：根据 React 文档，React 可能将 shouldComponentUpdate 视做提示而不是严格的根据它的返回结果决定是否执行，也就是说可能出现 shouldComponentUpdate 返回 false，但是还是发生重新渲染。

如前所述，该方法可能有有问题。React 官方提供了另一个解决办法，所以如果发现某个组件慢，可以使用React.PureComponent 替代 React.component，它将对 props 和 state 提供一个浅层对照。

componentWillUpdate（即将移除）

该方法在被渲染前调用。componentWillUpdate 在新的 props 进入组件或者 state 改变的时候调用。在该方法中不能调用 setState

常见的用例有：

  • 根据 state 的变化设置变量
  • 派发事件
  • 开始动画

注：componentWillUpdate 即将被移除，此方法可以替换为 componentDidUpdate()。如果你在此方法中读取 DOM 信息（例如，为了保存滚动位置），则可以将此逻辑移至 getSnapshotBeforeUpdate() 中。

getSnapshotBeforeUpdate()

getSnapshotBeforeUpdate() 在最近一次渲染输出（提交到 DOM 节点）之前调用。它使得组件能在发生更改之前从 DOM 中捕获一些信息（例如，滚动位置）。**此生命周期的任何返回值将作为参数传递给 componentDidUpdate()**。

此用法并不常见，但它可能出现在 UI 处理中，如需要以特殊方式处理滚动位置的聊天线程等。应返回 snapshot 的值（或 null）。

componentDidUpdate

componentDidUpdate() 会在更新后会被立即调用

componentWillUnmount

在卸载，销毁组件之前调用该方法。在卸载组件时候不能设置 State

  • 使用该方法清理 actions
  • 删除在 componentDidMount 或其他地方添加的事件监听
  • 断开网络请求
  • 清空计时器
  • 清理在 componentDidMount 中创建的 DOM 元素
