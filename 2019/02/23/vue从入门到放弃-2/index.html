webpack 作为一个集前端自动化、模块化、组件化于一体的可拓展系统，你可以根据自己的需要来进行一系列的配置和安装，最终实现你需要的功能并进行打包输出，在 Vue 的项目中，webpack 同样处理如打包压缩、异步加载、模块化管理等等。

webpack 在 CLI 3 中的应用

如果你使用过 vue-cli 2.x，那么你应该了解其构建出的目录会包含相应的 webpack 配置文件，但是在 vue-cli 3.x 中你却见不到一份关于 webpack 的配置文件，即你无需配置 webpack 就可以运行项目，并且它提供了一个 vue.config.js 文件来满足开发者对其封装的 webpack 默认配置的修改

vue.config.js 的配置介绍

下面了介绍一下vue.config.js 的配置项的功能和用途：

  • baseUrl: 项目根目录,其改变的其实是 webpack 配置文件中 output 的 publicPath 项;
  • outputDir: 配置大包输出的文件夹，这其实配置了 webpack 配置中 output 下的 path 项，修改了文件的输出路径, 当运行 你拼命 build 时，就会创建此文件夹并放入打包好的文件;
  • assetsDir: 放置生成的静态资源 (js、css、img、fonts) 的 (相对于 outputDir 的) 目录。;
  • lintOnSave: 这个值会在 @vue/cli-plugin-eslint 被安装之后生效，在开发环境下通过 eslint-loader 在每次保存时 lint 代码；
  • transpileDependencies: 默认情况下 babel-loader 会忽略所有 node_modules 中的文件。如果你想要通过 Babel 显式转译一个依赖，可以在这个选项中列出来。
  • productionSourceMap: 用于设置是否为生产环境构建生成 source map，一般在生产环境下为了快速定位错误信息，我们都会开启 source map,该配置会修改 webpack 中 devtool 项的值为 source-map。
  • chainWebpack: 是一个函数，会接收一个基于 webpack-chain 的 ChainableConfig 实例。允许对内部的 webpack 配置进行更细粒度的修改,该插件可以让我们能够使用链式操作来修改配置
  • configureWebpack:如果这个值是一个对象，则会通过 webpack-merge 合并到最终的配置中。如果这个值是一个函数，则会接收被解析的配置作为参数。该函数及可以修改配置并不返回任何东西，也可以返回一个被克隆或合并过的配置版本。
  • css: 默认情况下，只有 .module.[ext] 结尾的文件才会被视作 CSS Modules 模块。设置为 true 后你就可以去掉文件名中的 .module 并将所有的 .(css|scss|sass|less|styl(us)?) 文件视为 CSS Modules 模块。
  • devServer: 用于配置 webpack-dev-server 的行为，使得我们可以对本地服务器进行相应配置，我们在命令行中运行的 npm serve 对应的命令 vue-cli-service serve 其实便是基于 webpack-dev-server 开启的一个本地服务器，
