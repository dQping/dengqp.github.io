pointer-events: none 的应用场景

2019-03-06 | pointer-events | css
目录
  1. 实际应用场景

大家都知道 input[type=text|button|radio|checkbox] 支持 disabled 属性，可以实现事件的完全禁用。如果其他标签需要类似的禁用效果，可以试试 pointer-events: none 这个神奇的属性。

借用 CSS3 pointer-events:none 应用举例及扩展 的总结来说：

pointer-events: none 顾名思义，就是鼠标事件拜拜的意思。元素应用了该 CSS 属性，链接啊，点击啊什么的都变成了 “浮云牌酱油”。pointer-events: none 的作用是让元素实体 “虚化”。例如一个应用 pointer-events: none 的按钮元素，则我们在页面上看到的这个按钮，只是一个虚幻的影子而已，您可以理解为海市蜃楼，幽灵的躯体。当我们用手触碰它的时候可以轻易地没有任何感觉地从中穿过去。

实际应用场景

1、 表单 input 输入框上的内容，挡住了输入框被点击而获得焦点，这时候可以对输入框上方的元素设置 pointer-events:none ，使用场景如下：

  • 表单 input 输入框如果内容校验的报错是放在这个输入底部，这部分报错信息会遮住输入框的一部分区域，这部分区域被点击的时候，点击到的是放报错信息的元素而不是 input，影响了正常的用户体验
  • 需要 js 动态修改表单 input 的 placeholder 提示，但是在安卓手机上改 placeholder 不生效，这时候酒会用假的 placeholder 代替显示，这个假的 placeholder 元素会同上情况一样遮盖表单。
