生命周期钩子函数引起的错误

Vue 项目运行报错以下错误，但功能还是目前来说还是可以使用，从报错的 handlers[i].call 入手查找原因–这个错误是，调用相关的生命周期钩子函数引起来的错误。去找相应报错页面，查看页面相关的生命周期钩子函数 是否有 声明了未定义方法 或是 只声名了钩子函数，把钩子删除，或是把你定义的方法写完全了就可以了。

外部 SCSS 文件中图片路径的 loader 解析错误

在 Vue 中引用外部 SCSS(SCSS 里面使用了图片)时，loader 会报错，原因是：某 _.scss 文件引用了其它目录的图片文件，在 _.vue 中使用 scss 时，用 scss 的语法引入了该 *.scss 文件，最终 loader 解析图片路径出错，报错信息如下：

解决方法：借助 webpack 的~前缀

vue-cli3 一直运行/sockjs-node/info?t= 解决方案

首先 sockjs-node 是一个 JavaScript 库，提供跨浏览器 JavaScript 的 API，创建了一个低延迟、全双工的浏览器和 web 服务器之间通信通道。

  • 服务端：sockjs-node（https://github.com/sockjs/sockjs-node）
  • 客户端：sockjs-clien（https://github.com/sockjs/sockjs-client）
  • 如果你的项目没有用到 sockjs，vuecli3 运行 npm run serve 之后 network 里面一直调研一个接口：http://localhost:8080/sockjs-node/info?t=1462183700002
