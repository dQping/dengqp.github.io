数组和对象的拷贝和合并

javascript 中分为基本数据类型和引用数据类型，数组和对象都属于引用数据类型，所以在对象的拷贝和合并时，需要考虑深拷贝和浅拷贝问题，以及操作是否会污染原数据。在实际写代码中，有时会因为忽略了数组和对象的深拷贝和浅拷贝问题而出现 bug，往往到了调试中才发现问题，所一本篇对拷贝和合并常用的一些操作方法做小小的总结，也给自己加深一下记忆。

基本数据类型和引用数据类型的存储：
- 基本类型--名和值存储在栈内存中；
- 引用数据类型--名存在栈内存中，值存在于堆内存中，但是栈内存会提供一个引用的地址指向堆内存中的值；

拷贝

拷贝是让一个新变量等于一个原有的变量，如果是基本数据类型，let a = b，那么拷贝的就是数据本身，相当于有了 a、b 两份数据，数据之间的修改互不干扰；如果是引用数据类型，用 let a = b 的形式拷贝的就是数据 a 的引用地址，数据只有一份，a 、b 都是指向这个数据的地址，所以不管是针对 a 或 b 的操作都会修改同一份数据。

浅拷贝

拷贝的是数据的引用地址，因此修改拷贝后的对象会对原数据造成修改，这种拷贝方式会污染原数据。

深拷贝

拷贝的是数据本身，因此修改拷贝后的对象不会对原数据造成修改，这种拷贝方式不会污染原数据。

合并

数组合并常用的方法

- push(item1,.....,itemX): 改变原数组，返回改变后的数组长度
- unshift(item1,.....,itemX): 改变原数组，返回改变后的数组长度
- splice(index,howmany,item1,.....,itemX): 改变原数组，返回被删除的数组项，如果没有删除的项，返回[]

想要往数组中添加值，我们常用的是 push() 或 unshift() ; - 扩展运算符 … : [… arr1,arr2] 拷贝合并，返回新数组;

如果想要在合并完之后去除数组的重复项呢?

对象合并常用的方法

- Object.assign(target, obj1, obj2， obj3……) : 有重复的值时，后面的值会覆盖前面的值，属于深层覆盖，不管对象的深度多深，相同值都会被后面的值覆盖。
- 扩展运算符 ... : 返回新对象,有重复的值时，后面的值会覆盖前面的值，属于深层覆盖，不管对象的深度多深，相同值都会被后面的值覆盖。
