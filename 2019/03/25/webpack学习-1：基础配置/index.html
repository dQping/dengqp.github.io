样式打包

css 打包

要从 JavaScript 模块中导入 CSS 文件，您需要在模块配置中安装并添加 style-loader 和 css-loader

  • css-loader 会帮我们分析出几个 css 文件之间的关系，最终帮我们合并成一个 css 文件
  • style-loader 在得到 css-loader 生存的内容后，会把这段内容通过<style>标记将插入到 html 文件的<head>中。

loader 配置:

当运行该模块时，带有字符串化 css 的<style>标记将插入到 html 文件的<head>中。

sass 打包

sass-loader 需要 webpack 作为 peerDependency，它要求你自己安装 Node Sass 或 Dart Sass。这允许您控制所有依赖项的版本，并选择要使用的 Sass 实现。

  • sass-loader 将 Sass 编译为 CSS

loader 配置：

importLoaders：2 指定文件中通过 @import 引入的 sass 文件 也要通过 postcss-loader 和 sass-loader 的处理

less 打包

首先，需要安装 less-loader。less-loader 需要 less 作为 peerDependency。因此，您可以准确地控制版本。

  • less-loader 将 Less 编译为 CSS

importLoaders：1 指定文件中通过 @import 引入的 less 文件 也要通过 less-loader 的处理

添加浏览器厂商前缀

在写样式时，为了更好的兼容，一些 css 新属性我们会加上浏览器厂商浅醉，例如 transform: translate(100px, 100px)，我们可以通过配置，在打包的时候自动加上前缀。

首先，需要安装 postcss-loader，和 autoprefixer 插件

在目录中使用 postcss.config.js 文件。配置会查找从 path.dirname（文件）开始，然后向上遍历文件树，直到找到配置文件。

打包字体文件
