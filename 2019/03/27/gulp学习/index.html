gulp 中使用的是 node 的 stream（流） , 是以 stream 为媒介，先读取需要的 stream , 通过 pipe() 方法倒入到想要的地方，比如插件等等，经过插件处理的流可以继续导入到其他地方，包括写入文件中

前期

gulpfile.js

在项目中新建 gulpfile.js 文件用于 gulp 构建编写

流程： gulp.src() > pipe() > ….. > pipe() > dest()

路径匹配规则：

- * ： 匹配 0 或多个字符 ，例如 a , abc
- ** ： 匹配 0 或多个目录 ，例如 a/ , a/b/cc
- ? ： 匹配 1 个字符，例如 a/ , a/b/cc
- [...] ： 匹配 括号内任意 1 个字符，例如 a/ , a/b/cc
- !(p1|p2|p3) ： 排除法匹配
- ?(p1|p2|p3) ： 匹配括号中给定的任意模式 0 次或 1 次
- +(p1|p2|p3) ： 匹配括号中给定的任意模式至少 1 次
- *(p1|p2|p3) ： 匹配括号中给定的任意模式 0 次或多次
- @(p1|p2|p3) ： 匹配括号中给定的任意模式 1 次

API

- gulp.src(globs[, options]) 获取流，即读取我们要操作的文件；
   globs：Srting 格式，文件匹配模式，多个匹配模式可用数组
   options：Object 格式，一般用不到

- gulp.dest(path[, options]) 写文件；
   globs：Srting 格式，写入文件的路径
   options：Object 格式，一般用不到

- gulp.task(name[, deps], fn) 定义任务；
   name：Srting 格式，任务名字
   deps：一个包含任务列表的数组，这些任务会在你当前任务运行之前完成
   fn：该函数定义任务所要执行的一些操作
   注： 执行多个任务，可通过定义一个空任务，将要执行的多个任务当作依赖，例 如 gulp.task('default',['one','two','three'])

gulp 插件

- gulp-rename : 重命名
- gulp-uglify : JS文件压缩
- gulp-minify-css : CSS文件压缩
- gulp-minify-html : html文件压缩
- gulp-imagemin : 图片文件压缩
- gulp-jshint : JS代码检查
- gulp-concat : 文件合并
- gulp-less / gulp-sass : Less 和 Sass 的编译
- gulp-livereload : 自动刷新
- gulp-load-plugins : 自动加载插件
- gulp-order : 按制定顺序重新排列文件流
- gulp-spritesmith : 生成雪碧图
