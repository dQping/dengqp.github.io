Code Splitting

当我们在业务中使用一些第三方库的时候，最终打包成一个 js 文件，有可能会带来一些问题：

  • 问题 1: main.js 会比较大
  • 问题 2: 第三方库内容一般是不会被修改的，但业务代码会修改，每次修改业务代码，用户需要重新加载所有 js 的代码

webpack 中代码分割实际上底层使用了 SplitChunksPlugin 这个插件,webpack 中已经安装，不需要额外在安装。

chunk 是什么？就是代码片段，webpack 打包过后，dist 中的每一个 js 文件都是一个 chunk.

SplitChunksPlugin 的一些参数详解：https://webpack.js.org/plugins/split-chunks-plugin

如果不配置 splitChunks，那就按默认配置打包，默认配置如下：

  • chunks: 值 async,all,inital,表示对同步、所有、异步代码进行分割，chunks 和 cacheGroups 是配合使用的
  • minSize: 只有当引入的库大于 minsize 设置的值 kb，才会进行代码分割。
  • maxSize: 可配可不配，当引入的库大于 maxsize 设置的值 kb，会尝试进行二次拆分。
  • minChunks: 设置打包生成的文件中，当一个模块至少多少个文件使用，才会对它进行代码分割。
  • maxAsyncRequests:异步，设置为 5 表示页面同时加载的模块数最多是 5 个，如果超过这个值，就不进行代码分割了。
  • maxInitialRequests:同步，页面入口 html 最多加载的文件数量。
  • automaticNameDelimiter: 生成文件名使用的链接符。
  • name: 是否打包生成的文件命名。
  • cacheGroups: 根据 cacheGroups 的分组来决定分割出来的代码方法哦哪个文件中去。priority的值越大，组的优先级就越高。reuseExistingChunk 复用打包过的模块。

可以为每个 cacheGroups 的组再配置一个 filename,这样分到同一个组的代码会放入同一个命名的文件中。

Lazy Loading

懒惰或“按需”加载是优化网站或应用程序的好方法。这种做法主要涉及在逻辑断点处拆分代码，然后在用户完成需要或将需要新代码块的操作后加载它。这可以加快应用程序的初始负载并减轻其总体重量，因为某些块甚至可能永远不会被加载。

懒加载就是指通过 import 来异步的加载一个模块，但是具体什么时候去加载这个模块，要看代码怎么去写，什么时候真正的执行 import 这个语法的时候，代码才会真正的被载入。

懒加载能够提高我们页面的加载速度，vue, react, angular 等框架的路由都是使用懒加载模式实现的。

preloading 和 prefetching

查看代码使用率：我们打开控制台，在浏览器控制台，点击 command + shift + p, 输入 Show Coverage, 点击左侧录制的圆点按钮，然后刷新页面，可以看到页面所加载的文件的代码使用率。可以查看当前文件中用到的部分和没有用到的部分。

webpack 4.6.0+ 增加了对预取和预加载的支持。 在声明导入时使用一些内联指 ​​ 令允许 webpack 输出“Resource Hint”，它告诉浏览器： prefetch：将来某些导航可能需要资源 preload：当前导航期间可能需要资源，通过这种方式提高网站的访问速度。

preloading 预加载的块具有中等优先级并立即下载,预加载的块开始与父块并行加载。,prefetching 会等到主业务流程加载完成后，页面带宽空闲的时候，会自动去加载某些可能需要的资源。

webpack 与浏览器缓存

webpack 打包输出 output 中支持为文件名配置 contenthash 串，在每次更改代码打包上线时，可以防止用户浏览器缓存而不去请求最新文件的问题。contenthash 会根据打包生产的文件内容生产 hash 串，文件内容不变，hash 值就不会变

打包分析

可以通过运行 webpack –profile –json> stats.json 生成所需的 JSON(stats.json)文件，通过此文件进行分析，官方提供网址:https://github.com/webpack/analyse

还有社区提供的几种分析工具：

webpack-chart:webpack 统计信息的交互式饼图。

webpack-bundle-analyzer:一个插件和 CLI 实用程序，它将 bundle 内容表示为方便的交互式可缩放树形图。

webpack-visualizer:可视化分析生成的软件包以查看哪些模块占用空间以及哪些模块可能重复。

webpack bundle optimize helper:此工具将分析我们项目的捆绑包，并提供有关改进措施的可操作建议，以减少捆绑包的大小。
