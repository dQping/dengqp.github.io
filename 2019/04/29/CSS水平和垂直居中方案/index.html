CSS水平和垂直居中方案

我们在网页布局的时候，经常会碰到需要居中的情况，那下面就来总结一下有哪几种目前比较常用的居中方案，它们的优点和缺点分别又是什么。

水平居中

方法 ①

（父元素）text-align，（子元素）inline-block

这个是比较传统的做法，而且代码量也相对比较少，它的特点是可以很好地兼容 IE6 等低版本浏览器。至于缺点也比较明显，是需要把父元素和子元素都做成一个整体，并且上同时设置 CSS

优点：兼容性好，支持低版本浏览器，支持居中多个子元素的情况
缺点：需要同时在父元素和子元素上设置
适用场景：需要兼容低版本浏览器时使用

方法 ②

（子元素）display:table;margin:auto;（左右 auto 即可）

这个方法利用 display:table 这个样式把子元素转化成类表格元素，从而可以用 margin:auto 来自动居中。方案的优点是只需要在子元素上加 CSS，但 display:table 在 IE8 以上浏览器才有效果，所以本方法只适合用在较高版本浏览器中。

优点：只需要在子元素设置。
缺点：不兼容 IE6 和 IE7
适用场景：子元素数量多，不方便修改父元素的属性，对浏览器版本要求相对较低时使用

（子元素）display:block;margin:0 auto;（左右 auto 即可）

没有以上兼容问题，基本场景都适用

方法 ③

(父元素）position:relative，（子元素）absolute+left+transform（translateX(-50%)）/margin-left

这个的方法看起来就比较现代化了。没错，它的核心是利用了 CSS3 里的水平偏移，由于设置了 absolute，所以对其他元素基本不会造成影响

优点：不影响其他元素
缺点，transform 兼容性

以下这种方法不会有兼容问题，但必须知道子元素的宽度

方法 ④

flex+justify-content/margin

这个方法利用了 display:flex 对 Grid 操作的强大，分别有两种设置方法。

一种是（父元素）display:flex;justify-content:center;

优点：只设父元素,适用多个子元素
缺点：flex 兼容性差，而且比较耗资源

另一种是（父元素）display:flex;（子元素）margin:0 auto;

优点：代码简单
缺点：污染父元素，flex 兼容性问题

垂直居中：

方法 ①

（父）table-cell + vertical-align

优点：兼容性比较好，换成 table 可以兼容 IE6,7
缺点：table-cell 会不能和 float、position:absolute 同时设置。（只能在外 parent 外套一层 div.wrap 才能设置 float）

方法 ②

（父）position:relative；（子）absolute +top+ transform/margin

这个方法和水平居中的 left+transform 类似，这里就不再赘述了
优点：基本只设置子元素，不影响其他元素
缺点：transform 兼容性问题

方法 ③

（父）flex + align-items

方法和水平居中的 flex 很类似吧
优点：只要设置父元素
缺点：flex 和 align-items 的兼容性
