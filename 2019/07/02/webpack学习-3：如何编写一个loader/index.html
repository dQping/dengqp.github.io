webpack学习-3：如何编写一个loader

loader 实际上就是一个导出为函数的 JavaScript 模块。loader runner 会调用这个函数，然后把上一个 loader 产生的结果或者资源文件(resource file)传入进去。函数的 this 上下文将由 webpack 填充，并且 loader runner 具有一些有用方法，可以使 loader 改变为异步调用方式，或者获取 query 参数。

注意：通过 module.exports 导出的函数不能用箭头函数，因为 webpack 在调用 loader 的时候，会改变函数执行的上下文 this 指向，由 webpack 填冲的 this 具有一些有用方法和属性

同步 loader

第一个 loader 的传入参数只有一个：资源文件(resource file)的内容。compiler 需要得到最后一个 loader 产生的处理结果。这个处理结果应该是 String 或者 Buffer（被转换为一个 string），代表了模块的 JavaScript 源码。另外还可以传递一个可选的 SourceMap 结果（格式为 JSON 对象）。

返回多个处理结果

如果是单个处理结果(例如需要返回 SourceMap 结果的时候)，可以在同步模式中直接返回。如果有多个处理结果，则必须调用this.callback()

一个可以同步或者异步调用的可以返回多个结果的函数。预期的参数是：

  • 第一个参数必须是 Error 或者 null

  • 第二个参数是一个 string 或者 Buffer,就是文件内容处理后的结果

  • 可选的：第三个参数必须是一个可以被这个模块解析的 source map。

  • 可选的：第四个选项，会被 webpack 忽略，可以是任何东西（例如一些元数据）。

    可以将抽象语法树(abstract syntax tree - AST)（例如 ESTree）作为第四个参数（meta），如果你想在多个 loader 之间共享通用的 AST，这样做有助于加速编译时间。

    如果这个函数被调用的话，你应该返回 undefined 从而避免含糊的 loader 结果。

异步 loader

在异步模式中，必须调用 this.async()，来指示 loader runner 等待异步结果，它会返回 this.callback() 回调函数，随后 loader 必须返回 undefined 并且调用该回调函数。

自己编写 loader 的业务场景：

  1. 做代码异常监控：需要对业务代码加一些 try…catch 这样的异常捕获，可以编写 loader ，对代码进行分析，添加异常捕获代码
  2. 多语言版本适配：在 loader 中分析代码，获取 node 的全局变量，通过全局变量判断语言版本，进而显示对应的语言
  3. ……

如果我们发现我们的源代码需要做一些包装，这个时候，就可以考虑是否使用 loader。更多配置参考官方文档:https://webpack.docschina.org/api/loaders/
